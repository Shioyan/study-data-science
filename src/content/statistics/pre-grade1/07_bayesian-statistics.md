## 頻度論とベイズ論の考え方の違い

統計的推論には2つの根本的に異なるアプローチがある：

- **頻度論（Frequentist）**：確率を「試行を無限回繰り返したときの相対頻度」として定義する。パラメータは未知の固定値であり、データが確率変数。95%信頼区間は「同じ手順で100回実験すれば95回は真の値を含む区間が得られる」と解釈する。
- **ベイズ論（Bayesian）**：確率を「不確実性の度合い（主観的信念）」として定義する。パラメータそのものが確率分布を持つ確率変数であり、データを観測するたびにその分布を更新する。

## ベイズの定理

ベイズ統計の核心はベイズの定理である：

$$P(\theta \mid \mathbf{x}) = \frac{P(\mathbf{x} \mid \theta) \cdot P(\theta)}{P(\mathbf{x})}$$

統計的文脈では各項に名前がある：

- $P(\theta)$：**事前分布（prior）** — データ観測前のパラメータに関する信念
- $P(\mathbf{x} \mid \theta)$：**尤度（likelihood）** — パラメータが $\theta$ のとき観測データが得られる確率
- $P(\theta \mid \mathbf{x})$：**事後分布（posterior）** — データを観測した後の更新された信念
- $P(\mathbf{x})$：**周辺尤度（evidence）** — 正規化定数（$\theta$ に依存しない）

よって：**事後分布 $\propto$ 尤度 $\times$ 事前分布**

## ベイズ更新の例：コイン投げ

コインの表が出る確率 $\theta$ を推定する問題を考える。$\theta$ に一様事前分布 $\text{Beta}(1,1)$ を設定し、$n$ 回投げて $k$ 回表が出た場合、事後分布は：

$$\theta \mid k, n \sim \text{Beta}(k+1, n-k+1)$$

となる。事後分布の期待値は $(k+1)/(n+2)$ であり、データが蓄積されるほど尤度が支配的になり最尤推定量 $k/n$ に近づく。これが**ベイズ更新（Bayesian updating）**の直感的な例である。

## 共役事前分布

尤度関数と事前分布を掛け合わせたとき、事後分布が事前分布と同じ分布族に属する場合、その事前分布を**共役事前分布（conjugate prior）**という。計算が解析的に行え便利である：

| 尤度 | 共役事前分布 |
|------|-------------|
| 二項分布 | ベータ分布 |
| ポアソン分布 | ガンマ分布 |
| 正規分布（μ未知） | 正規分布 |

複雑なモデルでは共役関係が成立せず、MCMC（マルコフ連鎖モンテカルロ法）などの数値的手法で事後分布を近似する。

## ベイズ信用区間（Credible Interval）

事後分布から直接確率的に解釈できる区間推定を**ベイズ信用区間（credible interval）**という。最も一般的な形式は**HDI（Highest Density Interval）**や同等確率区間で、「パラメータが $[a,b]$ に含まれる確率が 95%」という直感的な解釈が可能である。

これは頻度論の信頼区間（「同じ手順を繰り返せば95%の確率で含む」という反直感的な解釈）とは根本的に異なる。

**落とし穴**：事前分布の選択がベイズ推論の結果に影響する。情報事前分布（informative prior）はサンプルサイズが小さいときに大きな影響を持つ。「無情報事前分布を使えば客観的」という誤解があるが、完全に客観的な事前分布は存在せず、無情報事前分布も一種の仮定であることを認識すべきである。

## Pythonでの活用例

```python
import pymc as pm
with pm.Model() as model:
    theta = pm.Beta('theta', alpha=1, beta=1)  # 事前分布
    obs = pm.Binomial('obs', n=10, p=theta, observed=7)  # 尤度
    trace = pm.sample(1000, return_inferencedata=True)
```
