## 2値アウトカムの予測問題

「メールはスパムか否か」「疾患を発症するかどうか」など、目的変数が0または1の2値をとる場合、通常の線形回帰は予測値が $[0,1]$ の範囲を超えてしまい確率として解釈できません。この問題を解決するのがロジスティック回帰です。

なぜ線形回帰がダメかというと、確率は $[0,1]$ の範囲に収まる必要があるのに、線形モデルは $-\infty$ から $+\infty$ の範囲の値をとるからです。また、$y$ が0/1のときの残差が正規分布に従わないため、最小二乗法の前提も崩れます。

## ロジット変換とシグモイド関数

事象の発生確率を $p$ とすると、**オッズ**は $p/(1-p)$ であり、そのログをとった**ロジット**は：

$$\text{logit}(p) = \ln\frac{p}{1-p}$$

ロジット変換により確率 $p \in (0,1)$ が実数全体 $(-\infty, +\infty)$ に写像されます。逆変換（確率に戻す関数）が**シグモイド関数**です：

$$p = \sigma(z) = \frac{1}{1 + e^{-z}}$$

### シグモイド関数のグラフと特性

シグモイド関数はS字曲線を描き、以下の特性を持ちます：

- $z \to -\infty$ で $p \to 0$、$z \to +\infty$ で $p \to 1$
- $z = 0$ で $p = 0.5$（ちょうど確率0.5を通過）
- 常に $(0,1)$ の範囲に収まるため確率として適切に解釈できる
- 滑らかで微分可能（最尤推定の数値最適化に適している）

```python
import numpy as np
import matplotlib.pyplot as plt

z = np.linspace(-6, 6, 300)
p = 1 / (1 + np.exp(-z))
plt.plot(z, p)
plt.axhline(0.5, linestyle='--', color='gray')
plt.xlabel('z（線形予測子）')
plt.ylabel('p（確率）')
plt.title('シグモイド関数')
plt.show()
```

## ロジスティック回帰モデルの定義

説明変数 $\mathbf{x} = (x_1, \ldots, x_k)$ のもとで $y=1$ となる確率を：

$$P(y=1 \mid \mathbf{x}) = \frac{1}{1 + \exp(-(\beta_0 + \beta_1 x_1 + \cdots + \beta_k x_k))}$$

と定義します。これはロジットが線形予測子に等しい、すなわち：

$$\ln\frac{p}{1-p} = \beta_0 + \beta_1 x_1 + \cdots + \beta_k x_k$$

という構造を持つ**一般化線形モデル（GLM）**の一種です。リンク関数がロジット関数、確率分布がベルヌーイ分布（二項分布の特殊ケース）です。

## オッズとオッズ比の直感的説明

### オッズとは何か

確率 $p = 0.75$（4回中3回成功）のとき、オッズは $0.75 / 0.25 = 3$ です。これは「成功が失敗の3倍起きやすい」ことを意味します。オッズは $[0, \infty)$ の値をとり、1より大きければ成功が優勢、1より小さければ失敗が優勢です。

### オッズ比とは何か

$x_j$ が1単位増加したとき、オッズは $e^{\beta_j}$ 倍になります。これを**オッズ比（OR）**といいます：

$$\text{OR}_j = e^{\beta_j}$$

- OR = 2 なら「その変数が1単位増えると、事象が起きるオッズが2倍になる」
- OR = 0.5 なら「オッズが半分になる（保護的な効果）」
- OR = 1 なら「影響なし」

**注意点**：オッズ比は「確率が2倍」ではなく「オッズが2倍」です。たとえば $p=0.1$ のとき OR=2 なら新しい確率は $p \approx 0.18$（約1.8倍）ですが、$p=0.5$ のとき OR=2 なら新しい確率は $p \approx 0.67$（1.3倍）です。イベント発生率が高い場合、オッズ比は相対リスク（リスク比）と大きく異なることに注意してください。

### 数値例

喫煙（0/1）の係数 $\beta = 1.2$ ならば $e^{1.2} \approx 3.32$ であり、喫煙者は非喫煙者に比べて疾患のオッズが約3.3倍高いことを意味します。

## 最尤推定法の概念

ロジスティック回帰のパラメータは最小二乗法ではなく**最尤推定法（MLE）**で推定します。対数尤度関数は：

$$\ell(\boldsymbol{\beta}) = \sum_{i=1}^n \left[ y_i \ln p_i + (1-y_i)\ln(1-p_i) \right]$$

これはバイナリ交差エントロピー損失の符号反転と同一であり、機械学習との橋渡しとなる重要な関係です。解析解は存在しないため、ニュートン-ラフソン法（IRLS: Iteratively Reweighted Least Squares）による数値最適化が行われます。

## ROC曲線とAUCの読み方

ロジスティック回帰の性能評価にはROC曲線とAUCが用いられます。分類閾値を変化させたとき、横軸に偽陽性率（FPR）、縦軸に真陽性率（TPR = 感度）をプロットした曲線がROC曲線です。

### ROC曲線の解釈

- 左上角に近い曲線ほど良いモデル
- 対角線（y=x）はランダム予測（コイン投げ）
- 完全予測モデルは左上角を通る（AUC=1.0）

### AUCの目安

| AUC値 | 評価 |
|-------|------|
| 0.5 | ランダム予測（無意味） |
| 0.6〜0.7 | 弱い識別能力 |
| 0.7〜0.8 | 許容できる識別能力 |
| 0.8〜0.9 | 良い識別能力 |
| 0.9以上 | 非常に良い識別能力 |

AUCは「ランダムに選んだ陽性例のスコアが陰性例のスコアより高い確率」として解釈することもできます。

## Pythonによる実装例

```python
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import (
    roc_auc_score, confusion_matrix, classification_report, RocCurveDisplay
)
from sklearn.model_selection import train_test_split
import matplotlib.pyplot as plt

# データ分割
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# モデルの学習
model = LogisticRegression(max_iter=1000)
model.fit(X_train, y_train)

# 予測確率と評価
y_prob = model.predict_proba(X_test)[:, 1]  # 陽性クラスの予測確率
auc = roc_auc_score(y_test, y_prob)
print(f'AUC: {auc:.3f}')

# 混同行列（閾値0.5での分類）
y_pred = model.predict(X_test)
print(confusion_matrix(y_test, y_pred))
# [[TN, FP], [FN, TP]] の形式

# ROC曲線のプロット
RocCurveDisplay.from_estimator(model, X_test, y_test)
plt.show()

# 係数とオッズ比
import numpy as np
odds_ratios = np.exp(model.coef_[0])
print("オッズ比:", odds_ratios)
```

## 混同行列の読み方

```
              予測: 陰性  予測: 陽性
実際: 陰性    TN（真陰性）  FP（偽陽性）
実際: 陽性    FN（偽陰性）  TP（真陽性）
```

- **感度（再現率）** = TP / (TP + FN)：陽性をどれだけ拾えたか
- **特異度** = TN / (TN + FP)：陰性をどれだけ正しく陰性と判定できたか
- **適合率（精度）** = TP / (TP + FP)：陽性と予測したもののうち本当に陽性の割合

## 適用の前提条件

1. **目的変数が2値**（多クラスの場合は多項ロジスティック回帰を使用）
2. **観測値が独立**
3. **説明変数と対数オッズの線形性**
4. **サンプルサイズが十分**（陽性・陰性ともに最低10〜20件以上が目安）
5. **多重共線性がない**（重回帰同様にVIFで確認）

## 実務での落とし穴

- **クラス不均衡（class imbalance）への注意**：陽性が1%などの場合、「すべて陰性と予測する」モデルでも精度99%になります。AUCに加えて適合率・再現率・F1スコアも確認してください。対処法として、クラス重みの調整（`class_weight='balanced'`）や過小サンプリング・SMOTE（オーバーサンプリング）があります。
- **外挿の問題**：説明変数の値が学習データの範囲外の場合、シグモイド関数の飽和域で確率が0か1に張り付き、識別に使えなくなります。
- **係数の解釈はオッズ比で行う**：生の係数 $\beta_j$ は対数オッズの変化量であり、直接「確率の変化量」として解釈してはいけません。
