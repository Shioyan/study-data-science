# Study Data Science — 個人用学習ウェブアプリ

## プロジェクト概要

データサイエンスおよび統計検定の勉強を効率的に進めるための**個人用ウェブアプリ**を開発する。

### 主な機能
1. **学習テキスト** — 概念の解説文と図解をセクションごとに提供
2. **クイズ（練習問題）** — 選択式・記述式の問題で理解度を確認
3. **解説フィードバック** — 不正解時に「なぜ間違えたか」を丁寧に解説
4. **復習リンク** — 関連するテキストセクションへのリンクで弱点を補強

## 学習対象範囲

### 統計検定
- 統計検定4級・3級・2級・準1級・1級
- 統計検定データサイエンス系（DS基礎・DS発展・DS Expert）

### データサイエンス
- 記述統計・推測統計
- 確率・確率分布（正規分布・二項分布・ポアソン分布 等）
- 仮説検定・区間推定
- 回帰分析（線形・ロジスティック）
- 機械学習の基礎（分類・回帰・クラスタリング）
- データ前処理・探索的データ分析（EDA）
- Pythonによるデータ分析（pandas・numpy・matplotlib・scikit-learn）

## アプリ構成（機能設計）

```
学習モード
  └── テキスト閲覧
        ├── セクション一覧（サイドバー）
        ├── 本文（Markdown → HTML レンダリング）
        └── 図解（静的画像 or インタラクティブ図）

クイズモード
  └── 問題表示
        ├── 選択肢（4択 or 正誤）
        ├── 回答 → 正解 or 不正解の判定
        ├── 不正解時：解説文の表示
        └── 不正解時：関連テキストセクションへのリンク

進捗管理
  └── 問題ごとの正解率・履歴（ローカルストレージ or DB）
```

## 技術スタック方針

> 具体的な技術選定はClaudeと相談しながら決める。以下は初期方針。

- **フロントエンド**: React（または Next.js）＋ Tailwind CSS
- **テキストレンダリング**: Markdown → MDX or react-markdown
- **図解**: Mermaid.js or D3.js（インタラクティブな図が必要な場合）
- **クイズデータ**: JSON or MDX ファイルで管理（DB不要なシンプル構成を優先）
- **状態管理**: ローカルストレージ（個人用のため認証不要）
- **ホスティング**: Vercel or GitHub Pages（静的サイトを優先）

## 開発方針

### シンプルさを優先する
- 個人用途のため、認証・サーバーサイド処理は最小限にする
- コンテンツ（テキスト・クイズ）はファイルで管理し、DBは極力使わない
- まず動くものを作り、後から機能を追加する

### コンテンツの品質
- テキストは「なぜそうなるか」の直感的な説明を優先する（公式の暗記でなく理解）
- 図解は概念の視覚化に使う（グラフ・フローチャート・数直線 等）
- クイズは統計検定の過去問レベルを意識して作成する

### クイズ設計ルール
- 1問につき必ず「解説文」と「関連テキストセクションへのリンク」を持たせる
- 不正解の選択肢（ひっかけ）にも解説を付ける（なぜそれが間違いかを明示）
- 難易度タグ（基礎 / 標準 / 応用）を付与する
